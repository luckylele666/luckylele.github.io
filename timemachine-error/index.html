关于 Time Machine 的迷惑……

本文最后更新于：June 11, 2022 pm

前天升级到 macOS Ventura 后非常难用，出现各种 bug，遂降级回 macOS Monterey，降级前想用 Time Machine 备份，可是提示「空间不足，无法备份」，无奈，直接降级。

成功恢复到 macOS Monterey 后再用 Time Machine 备份，可是依旧如此。就开始折腾，找资料，问客服，和客服在电话里沟通很久，021 开头的普通客服是外包的，高级客服才是 Apple 自己的，但是也看人，有些懂行有些也不行，高级客服上面还有专员和经理，应该是能在电话里接触到专员的，可是处理我 case 的这位说到她为止了，应该直接找工程部的那样处理这个技术问题会好一些。

从高级客服处弄到几个指令：

tmutil thinlocalsnapshots / 999999999999999 4（大概是这么个意思）

sudo defaults delete /Library/Preferences/com.apple.TimeMachine

tmutil listbackups|xargs basename -s .backup|tail -r|tail -n 3|xargs -I {} sudo tmutil delete -d /Volumes/TimeMachine/ -t {time}（这条是网友提供的,Volumes 填 SSD 名称，{} 里填时间，格式为 20220531221002，可以删掉最早 3 个备份）

上述一顿操作后依旧没有解决问题，又过了俩小时，居然莫名其妙的自动备份了，真是折腾和浪费时间，追求好的体验还是使用付费软件吧，例如 Carbon Copy Cloner 6，或者云端备份 Backblaze，节省时间珍惜生命。

另，如果觉得 Time Machine 的本地快照占空间，可以取消勾选 Time Machine 的「自动备份」选项来删除本地快照，释放空间[1]。Apple 好愚蠢的设定啊！
